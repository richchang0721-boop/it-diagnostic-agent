ENTRY // 網路診斷
能 ping 通閘道？
確認本機 L3 連線 → 依結果選擇路徑
否 → 本機問題
本機設定
IP / DNS / 閘道
DHCP 衝突或靜態設定錯誤
展開步驟
排查步驟
  • 執行 ipconfig /all 確認 IP 範圍
  • 釋放並重新取得：ipconfig /release → /renew
  • 確認預設閘道與子網路遮罩正確
  • 排除 IP 衝突：arp -a
  • 檢查網路卡驅動是否正常
是 → 往上排查
DNS 解析
nslookup / DNS 正常？
能解析網域名稱嗎？
展開步驟
  • nslookup google.com 確認 DNS 回應
  • 改用 8.8.8.8 測試是否 DNS Server 問題
  • 清除 DNS 快取：ipconfig /flushdns
  • 確認防火牆未封鎖 UDP 53

CCTV
CCTV 故障排除

展開
無影像 / 斷線
  • 確認 PoE Switch port 供電正常
  • Ping IP Camera IP 位址
  • 確認 NVR 頻道 IP 對應設定
  • ONVIF 協定版本相容性確認

  • 確認頻寬足夠（HD 約 2-4 Mbps/路）
  • H.264 → H.265 節省頻寬
  • QoS 設定優先 CCTV 流量
遠端存取問題
  • Port forwarding 80/554/8080
  • DDNS 設定確認
  • 建議使用 VPN 安全存取

無法開機

  • GPO 印表機部署設定確認
